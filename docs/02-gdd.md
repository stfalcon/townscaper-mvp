# Game Design Document (GDD)

## 1. Геймплейна петля (Core Loop)

Основна петля триває 3-5 секунд:

```
[Вибір кольору] → [Hover клітинки: preview-підсвітка] → [ЛКМ] →
[Будинок зʼявляється з bounce-анімацією] → [Сусідні тайли переформовуються] →
[Візуальна satisfaction] → повтор
```

Вторинні петлі:
- **Знесення-забудова:** ПКМ → видалити → поставити інший колір
- **Побудова башти:** клікати на верх існуючого будинку → додавати поверхи
- **Random city:** R → місто за 2 секунди → редагування

## 2. Керування (Controls)

### Desktop (основна платформа)

| Дія | Ввід | Примітка |
|-----|------|----------|
| Поставити будинок | ЛКМ по клітинці | placement mode (default) |
| Видалити будинок | ПКМ по будинку; АБО в Erase-mode — ЛКМ | для трекпадів без ПКМ |
| Mode toggle (Build/Erase) | клік по on-screen кнопці (не клавіатура) | **прихований до першого placement** — progressive disclosure |
| Обрати колір | клавіші `1`-`5` (`6` після unlock), клік по палітрі | |
| Обертати камеру | `Q` / `E` (90° snap, 200мс ease) | |
| Зум | колесо миші | clamp 0.5x-2x, spring clamp на межах |
| Згенерувати випадкове місто | клавіша `R` | stagger-анімація 1.5с |
| Очистити все | клавіша `C` + **утримати 1.5с** | hold-to-confirm, не Yes/No модалка |
| Показати help | клавіша `H` або кнопка `?` | |
| **Keyboard grid cursor** (для 12yo / a11y) | Arrow keys рухають ghost-cursor | WCAG 2.1.1 |
| **Place / Remove клавіатурою** | `Space` / `Delete` на позиції ghost-cursor | |

**Drag vs click:** клік = `pointerup` у межах **8px і 300мс** від `pointerdown`. Інакше = drag, ігнорується. Захист від дитячих micro-drag.

### Mobile (stretch goal)

| Дія | Ввід |
|-----|------|
| Поставити | tap |
| Видалити | long-press (≥500мс) або Erase-mode + tap |
| Обрати колір | tap по палітрі |
| Обертати/зумити | two-finger rotate / pinch |

## 3. Правила світу

### 3.1 Сітка
- **Тип:** квадратна 2D-сітка (координати `x`, `z`) + вертикальна вісь `y` (поверхи)
- **Розмір:** 30×30 клітинок, до 10 поверхів
- **Межі:** візуально — трава 30×30 тайлів, фон — мʼякий градієнт неба. Поза межами — не ставимо.
- **Порожня клітинка при hover:** світиться поточним кольором палітри з прозорістю 40%

### 3.2 Будинки / Тайли

**Типи (4 варіації для MVP):**

1. **`freestanding`** — будинок-одинак (0 горизонтальних сусідів): невеликий кубик з дашком-пірамідою
2. **`wall`** — стіна (2+ горизонтальних сусідів): простий кубик з рівним верхом
3. **`corner`** — кут (1 горизонтальний сусід): кубик з легким скосом
4. **`roof`** — дах (немає клітинки зверху): завжди має похилу або пласку верхівку замість звичайної

**Алгоритм вибору тайла:**
```
neighbors = count(north, south, east, west) не-null
hasAbove  = cell(x, y+1, z) != null

if !hasAbove:      → "roof"
else if n == 0:    → "freestanding"  (теоретично не буває якщо hasAbove, але safety)
else if n == 1:    → "corner"
else (n >= 2):     → "wall"
```

Переобчислення: при додаванні/видаленні клітинки → перераховуємо `tileType` для самої клітинки + 6 сусідів (4 горизонтальних + above + below).

**Вертикальна забудова:**
- Дозволена до `y = 9` (10 поверхів)
- Можна ставити в повітрі лише якщо є опора знизу АБО хоча б один горизонтальний сусід на тому ж рівні
- Інакше UI показує червону hover-підсвітку і клік ігнорується

### 3.3 Кольори
**5 основних кольорів** з палітри (див. Art Brief §3) + **1 bonus "Surprise" колір** розблоковується після 10 placements. Bonus — насичений помаранчевий `#FF6B3D`, single-reward для 7-річки щоб не застрявала у муті́д-палітрі.

Вибираються клавіатурою `1`-`5` (`6` після unlock) або кліком по палітрі внизу екрану. Поточний вибраний — підсвічений білим обрамленням 3px.

При unlock 6-го кольору: плавна fade-in 6-ї circle-button + sparkle effect 400мс + без модалки (тиха нагорода).

## 4. UX-флоу (перші 60 секунд) — v2 після UX-ревью

**Проблема v1:** 7-річка не здогадається перевести курсор на canvas. Pulse на клітинці невидимий поки курсор не там.

**Рішення:** screen-anchored великий SVG-arrow вказує на pulsing-cell + пульсація краю canvas.

| Час | Що відбувається |
|-----|-----------------|
| 0-0.5с | Порожнє поле, камера робить легкий пан-ін. Trava + небо |
| 0.5-1с | Край canvas отримує пульс 2px теплий білий — «інтеракція тут» |
| 1-2с | Fade-in **великий SVG-arrow** у центрі viewport, вказує на центр canvas. Під ним текст **«Тикни — побудуй!»** 28px |
| 2-∞с | Центральна клітинка (15, 0, 15) починає пульсувати білим outline |
| Будь-який pointermove над canvas | arrow fade-out 200мс (дитина зрозуміла) |
| First click на будь-яку клітинку | Будинок зʼявляється з bounce, залишки hint зникають |
| +3с після першого placement | Палітра легко пульсує 600мс — «спробуй інші кольори» |
| Після 5 будинків | Fade-in підказка у верхньому правому куті: **«R — ціле місто!»** 18px, зникає через 8с |
| Після 10 будинків | **Unlock 6-го кольору** — sparkle-effect + fade-in 6-ї кнопки у палітрі, без модалки |
| Після першого placement | **Mode toggle (Build/Erase) стає видимим** у top-right (progressive disclosure) |

**Ніяких модалок, ніяких tutorial-екранів.** Навчання через саму гру.

### Erase-mode silent-failure protection (NEW)

Якщо дитина випадково ввімкнула erase і клікає по порожньому полю:
- Inline toast біля курсора: **«Ти стираєш 🧽»** 18px, контрастний фон
- Mode-button робить shake-animation 300мс
- Через 1с toast fade-out
- Після 3-х таких no-op кліків підряд — автоматичний switch назад у Build mode + toast «Будуємо далі!»

### Micro-celebrations (NEW — для 10+ хвилинної retention)

- Після **10 будинків:** sparkle + «Яке місто!» текст 1с (+ unlock 6-го кольору)
- Після **першої башти 3+ поверхи:** sparkle + «Ого, вежа!»
- Після **30 будинків:** sparkle + «Мегаполіс!»

Без score, без ачівок, без модалок — тільки візуально-емоційні mini-wins для підтримки сесії.

## 5. Feedback / Juice

| Подія | Ефект | Тривалість |
|-------|-------|------------|
| Hover клітинки | outline fade in (поточний колір, alpha 0.4) | 100мс |
| Unhover | outline fade out | 80мс |
| Поставити | scale Y [0, 1.1, 1.0] easeOutBack(1.7) | 250мс |
| Видалити | scale Y [1.0, 1.1, 0] easeInQuad | 150мс |
| Переформування тайла | плавний morph (lerp 2 меші) | 150мс |
| Вибір кольору | button scale [1, 1.15, 1] easeOutQuad | 150мс |
| Обертання камери | yaw lerp easeInOutCubic | 200мс |
| Random city | cascade place з delay 40мс per cell | ~1500-2000мс total |
| SFX place (опц) | «pop» ~180Hz envelope | 80мс |
| SFX remove (опц) | «whoosh» descending | 150мс |

Принцип: **кожна дія дає миттєвий візуальний відгук ≤100мс.**

## 6. Режими

- **Sandbox** (default) — будуй необмежено
- **Random city generator** — клавіша R. Очищає поле, генерує 30-60 будинків BFS-алгоритмом з центру, stagger-анімація 40мс на клітинку. Кольори — когерентні (80% як у сусіда, 20% випадкових).

Інших режимів у MVP немає.

## 7. Win/Lose умови

**Немає.** Це sandbox-творчий режим, подібний до Minecraft Creative або цифрового LEGO.

Обґрунтування: ціль гри — **радість творчості**, не challenge. Додавання цілей (збудуй форму, набери очок) змінило б психологічний контракт з гравцем і створило б стрес.

## 8. Прогресія / Утримання

**MVP: немає.** Гра — «зайшов, побудував, вийшов».

Пост-MVP ідеї (не робимо зараз):
- Ачівки (поставив 50 будинків, збудував башту 10 поверхів)
- Нові кольори після N дій
- Pre-sets («будуй як у Італії», «японське селище»)
- Share через URL-hash з серіалізованим станом

## 9. Доступність

- **Контраст:** всі кольори палітри перевіряємо проти трави на WCAG AA (4.5:1 для тексту, 3:1 для UI)
- **Target size:** всі клікабельні UI ≥48×48 px (палітра — 56×56)
- **Типографіка:** body ≥18px, hint ≥16px, modal ≥16px, onboarding headline ≥28px
- **Клавіатурна навігація (WCAG 2.1.1):**
  - Tab через DOM UI (палітра → mode toggle → help)
  - `canvas` має `tabindex="0"`, візуальний focus ring через `:focus-visible`
  - Arrow keys рухають ghost-cursor по клітинкам сітки
  - Space = place, Delete = remove
  - Shortcuts 1-5/R/H/C/Q/E слухаються на document з early-return якщо focus у `<button>`/`<input>`
- **Кольори + форма:** тайли відрізняються не тільки кольором, а й типом (wall/corner/roof) — захист від дальтонізму
- **Мова:** українська UI. Тексти прості, уникаємо жаргону
- **Іконки:** inline SVG (не emoji) — гарантія рендерингу на Windows/Linux

## 10. Адаптація під вік

| Вік | Адаптація |
|-----|-----------|
| **7 років** | Build-only mode до першого placement (erase прихований). Onboarding arrow. Erase-toast при silent failure. Micro-celebrations. Hold-to-confirm для Clear. Help modal показує 2 прості іконки (не всі shortcuts). |
| **12 років** | Після першого placement весь UI доступний. Help modal має toggle «Показати всі команди» → keyboard shortcuts, Q/E camera, Random city. Keyboard grid cursor для тих хто хоче грати з клавіатури. |

**Help modal — 2 рівні:**
- Default для 7yo: 4 великі візуальні інструкції (клікни → постав, клікни правою → знеси, 1-5 → колір, R → місто)
- Toggle «Показати всі команди» для 12yo: повна таблиця shortcuts

**Не робимо окремих режимів/профілів** — один UI, прогресивне розкриття через help-toggle і unlock.
