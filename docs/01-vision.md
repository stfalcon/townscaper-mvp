# Vision / One-Pager

## Назва проєкту
**Cozy Town Builder** _(робоча назва)_

## Pitch
Casual sandbox-містобудівник у браузері для дітей 7-12 років. Клік — і зʼявляється будиночок, який сам вписується у місто. Без правил, без цілей, без програшу — тільки творчість. Один HTML-файл, відкривається в Chrome/Safari без інсталяції.

## Для кого
- **Первинна аудиторія:** діти 7 і 12 років (діти автора)
- **Вторинна:** батьки, які хочуть показати дітям можливості АІ у 2026
- **Платформа:** браузер desktop (Chrome 100+ / Safari 16.4+ / Firefox 108+). Планшет і touch — OUT з MVP (узгоджено — див. Скоуп OUT).

## Основний досвід (core fantasy)
Гравець за 30 секунд після старту відчуває: **«Я творець — моє місто народжується і саме стає красивим»**. Без навчання, без frustration, без блокуючих екранів.

Додатковий емоційний шар для дітей: **«тато це написав за вечір з АІ — значить і я зможу»**. Демо-ефект важливіший за глибину геймплею.

## Референси
- **Основний:** Townscaper (Oskar Stålberg) — настрій, палітра, satisfaction від сусідства тайлів
- **Додаткові:** Minecraft Creative (знайомий паттерн), Islanders (cozy placement), Block'hood (мʼяка ізометрія)
- **Не-референси:** бойовка, ресурс-менеджмент, таймери, tower defense, будь-який стрес

## Успіх MVP = ? (бінарні критерії)

Після ревізії success criteria переформатовані як **measurable бінарні** — можна поставити ✅/❌.

1. **[UX-7]** 7-річна дитина ставить ≥5 будинків без допомоги за ≤60 секунд від першого погляду на екран
2. **[UX-7]** Сесія гри 7-річки ≥5 хвилин (вимірюється timer у dev-консолі під час playtest)
3. **[UX-12]** 12-річна дитина за сесію ≥1 раз використовує Random city (R) і ≥1 раз використовує палітру
4. **[DEMO]** Гра запускається через `npx serve .` у Chrome на машині батьків ≤10 секунд від `cd` до першого clickable-стану
5. **[PERF]** 60 FPS при 500 будинках на Intel UHD 620 (типовий ноут 2018-2020) у Chrome
6. **[SAVE]** Місто з 50+ будинків відновлюється після F5 за ≤2 секунди, без втрат
7. **[DEMO]** Є готовий demo-script для показу дітям (див. T-DEMO у backlog)

_Критерії 1-2 — primary. Якщо failed — релізу немає. Критерії 3-7 — should-have._

## Скоуп — IN
- Квадратна сітка 30×30, до 10 поверхів висоти
- 5-кольорова палітра (теплі пастельні)
- ЛКМ — поставити, ПКМ — видалити, + on-screen mode toggle для 7-річок
- 4 варіації тайлів за сусідами: freestanding / wall / corner / roof
- Генератор випадкового міста (клавіша R) — «вау»-ефект
- Очистити все (клавіша C, з підтвердженням)
- Автозбереження у `localStorage`
- Scale-in/out анімації при постановці/видаленні
- Hover-підсвітка клітинки поточним кольором
- Допоміжний help (клавіша H або кнопка ?)

## Скоуп — OUT
- Irregular quad grid (Townscaper magic) — занадто складно для MVP
- Ручна бібліотека 200+ мешів — процедурні меші замість цього
- Multiplayer, cloud save, share-посилання
- Мобільний touch-ввід — stretch, якщо лишиться час
- Звукова система — stretch
- Ачівки, прогресія, розблокування
- Експорт у PNG / GIF

## Ризики

1. **Низька художня якість** без артиста → мітигація: fragment Y-gradient shader + сильна палітра + анімації + hemisphere light
2. **7-річка не знайде canvas / не зрозуміє erase-mode** → мітигація: onboarding arrow anchored до viewport center + toast-feedback у erase-mode + progressive disclosure (erase прихований до першого placement)
3. **Performance деградує** на Intel UHD (машини батьків) → мітигація: 4-пульний `InstancedMesh` + `instanceColor` + math-picking (DDA) без raycast по мешах
4. **Wow-ефект не спрацює** (діти звикли до Minecraft) → мітигація: Random City на R + 6-й bonus color що розблоковується після 10 будинків
5. **Кидки часу 1.5-2×** → мітигація: чесний 48h бюджет, hard-cut список якщо горить (див. нижче)

## Часовий бюджет

**~48 годин** розподілено на **5 спринтів по ~9-10 год** (6-8 вечорів).

_Важливо:_ після ревью 6 профільних агентів початкова оцінка 26h → 32h → **48h**. Типові MVP біжать 1.5-2× over budget при декомпозиції деталей; ця цифра вже враховує.

Розбивка:
- Sprint 1 (Infra + GameState + TileResolver): ~10h
- Sprint 2 (InstanceColor pools + Math picking + Place/Remove): ~12h
- Sprint 3 (Tile variations + Re-tile + Tests): ~10h
- Sprint 4 (UI + Palette + Save + Camera): ~9h
- Sprint 5 (Polish + Random city + Onboarding + Demo prep): ~7h

**Hard-cut list (якщо горить):**
